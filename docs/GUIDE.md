# 💹 주식 테마 분석기 (Stock Theme Analyzer) - 개발 가이드

이 문서는 프로젝트의 구조, 개발 원칙 및 협업을 위한 가이드라인을 담고 있습니다. AI 어시스턴트나 새로운 개발자가 프로젝트에 참여할 때 이 문서를 가장 먼저 참고하십시오.

---

## 1. 프로젝트 핵심 개념

본 프로젝트는 **"오늘의 핫 테마 종목 분석 및 수익성 검증 자동화"**를 목표로 합니다.

- **Data First**: 모든 데이터는 파이썬이 객관적으로 수집 및 분석합니다.
- **Premium UI**: 분석된 결과는 사용자에게 '프리미엄' 감성의 현대적인 대시보드로 제공됩니다.
- **Separation of Concerns**: 데이터 분석 엔진(Python)과 UI 렌더링 엔진(JS)은 완전히 독립되어 동작합니다.

---

## 2. 시스템 아키텍처 및 역할

프로젝트는 **데이터 엔진**과 **UI 엔진**으로 이원화되어 있습니다.

### 2.1 🐍 데이터 분석 엔진 (파이썬 / `main.py`)

- **수집 파이프라인 (2-Track)**:
  1. **단기 급등 테마주 추적**: 네이버 금융 테마 및 종목 수집 + `pykrx`를 통한 실시간 투자 지표 수집.
  2. **[AUTO] 국내 성장 저평가주 발굴**:
     - **GARP형 탐색**: 미래 성장 테마(AI, 로봇, 반도체, 에너지 등) 안에서 가격이 합리적이면서 시장 관심이 회복되는 종목을 우선합니다.
     - **과도한 소외주 제외**: 단순히 많이 빠진 종목이 아니라, 52주 고점 대비 하락폭이 과도하지 않고 최근 1개월 수익률, 거래대금, 20일/60일선 회복이 확인되는 종목을 선별합니다.
     - **성장 저평가 점수화**: 밸류에이션, 목표가 상승여력, 수급 회복, 이평선 회복, 유동성을 점수화하고 과도한 하락/약한 모멘텀은 감점합니다.
  3. **[AUTO] 미국 성장 저평가주 발굴**:
     - **GARP 필터**: PER/PBR이 과도하지 않으면서 Forward PE 개선, 매출 성장률, EPS 성장률 중 하나 이상이 확인되는 종목을 우선합니다.
     - **관심 회복 조건**: 1개월 수익률, 20일/50일선 회복, 거래량 회복, 주도 섹터 여부를 반영하여 너무 소외된 종목을 제외합니다.
     - **러셀 1000 테마 탐색**: S&P 500뿐만 아니라 **Russell 1000(미국 상위 1,000개 기업)**까지 범위를 넓혀 유망한 중견 성장주를 발굴합니다.
- **투자 지표/수익성 검증**:
  - **프리미엄 지표 산출 (Investing.com 스타일)**:
    - **상승여력(Upside)**: 증권사 목표주가(컨센서스)와 현재가를 비교하여 기대 수익률 산출.
    - **적정가치(Fair Value)**: 전문가들이 판단한 기업의 평균 목표가 표시.
    - **투자 등급**: 재무건전성(부채비율 등), 수익성(ROE 등), 성장성(영업이익률 등)을 분석하여 [최고/우수/보통/주의] 4단계 등급 부여.
    - **투자의견**: 애널리스트들의 종합적인 매수/매도 의견 수집.
  - **국내 적정가 수집 최적화**: 네이버 금융의 우측 사이드바(사이드 컨센서스) 및 하단 보고서를 교차 검증하여 중대형주의 목표주가 수집 성공률을 극대화함.
  - Open DART API 및 네이버 재무제표 스크래핑을 혼용하여 지표 수집(Fallback 로직 내장).
- **저평가 선별 기준**: 국내/미국 성장 저평가의 세부 조건과 점수화 방식은 `2.2 저평가 선별 기준`을 따릅니다.
- **성장 테마 자동 탐지 원칙**: 성장 키워드는 강제 조건이 아니라 안전장치입니다. 네이버 테마 상대강도, 선택적 뉴스/LLM 보조 점수, 최종 검증 조건을 함께 사용하며 검증을 통과한 테마만 `growth_focus`에 포함합니다.
- **출력**: 분석 결과를 `data.json` 파일로 생성.
- **알림**: 분석 완료 시 카카오톡 알림 메시지 전송 (`kakao_api.py`).

### 2.2 저평가 선별 기준

저평가 탭은 단순히 가격이 많이 빠진 종목을 찾는 기능이 아닙니다. 목표는 **성장 가능성이 있는 사업 영역 안에서 가격이 합리적이고, 시장 관심이 다시 붙기 시작한 종목**을 찾는 것입니다.

#### 성장 테마 자동 탐지

성장 테마는 완전 하드코딩하지 않습니다. 다만 자동 탐지의 노이즈를 줄이기 위해 키워드 기반 안전장치와 시장 데이터 기반 검증을 함께 사용합니다.

1. **기본 키워드 기반 유지**: AI, 로봇, 반도체, 전력, 데이터센터, 바이오, 우주, 방산 등 알려진 성장 키워드는 가산점으로만 사용합니다.
2. **자동 후보 테마 점수 추가**: 네이버 테마 상승률, 전체 테마 내 상대 순위, 강한 시장 관심도를 점수화합니다. 키워드에 없는 새 테마도 시장 강도가 충분하면 후보가 될 수 있습니다.
3. **뉴스/LLM 보조 점수 사용**: 뉴스나 LLM 기반 분류는 선택적 보조 점수입니다. 단독으로 테마를 채택하지 않고, 시장 점수와 함께 검증될 때만 반영합니다.
4. **검증 통과 테마만 포함**: 구경제 테마이면서 상승률이 약한 경우는 제외하고, 키워드 점수, 자동 시장 점수, 보조 점수 중 최소 하나가 충분한 테마만 `growth_focus`에 포함합니다.

현재 구현에서는 `ENABLE_THEME_CONTEXT_SCORE=true` 환경 변수를 설정한 경우에만 보조 문맥 점수를 반영합니다. 실제 뉴스/LLM API를 연결하더라도 이 점수는 보조 신호로만 사용해야 합니다.

#### 핵심 방향

- **저평가**: PER, PBR, 목표가 대비 상승여력으로 가격 부담을 확인합니다.
- **성장성**: 성장 테마, Forward PE 개선, 매출 성장률, EPS 성장률을 확인합니다.
- **관심 회복**: 최근 1개월 수익률, 이평선 회복, 거래량/거래대금 회복을 확인합니다.
- **소외주 제외**: 너무 오래 방치되었거나 52주 고점 대비 과도하게 하락한 종목은 제외합니다.

#### 국내 성장 저평가

국내 저평가는 미래 성장 테마 안에서 아래 조건을 점수화합니다.

1. 성장 테마 안에 포함될 것
2. PER 35 이하, PBR 4 이하일 것
3. 목표가 상승여력 12% 이상일 것
4. 52주 고점 대비 하락폭이 3%~50% 구간일 것
5. 최근 1개월 수익률이 -10%보다 양호할 것
6. 현재가가 20일선 또는 60일선 위로 회복했을 것
7. 최근 거래량이 20일 평균 대비 크게 죽지 않았을 것
8. 최소 거래대금 10억 원 이상, 우선 점수에는 30억 원 이상을 반영할 것

감점 조건은 다음과 같습니다.

- 52주 고점 대비 50% 이상 하락
- 최근 1개월 수익률 -10% 이하
- 상승여력이 80%를 넘는 과도한 목표가 괴리

#### 미국 성장 저평가

미국 저평가는 GARP 관점으로 아래 조건을 점수화합니다.

1. PER 45 이하, PBR 12 이하일 것
2. 시가총액 20억 달러 이상일 것
3. 평균 거래량 30만 주 이상일 것
4. Forward PE가 Trailing PE보다 낮거나, 매출 성장률 또는 EPS 성장률이 양수일 것
5. 목표가 상승여력 12% 이상일 것
6. 52주 고점 대비 하락폭이 3%~50% 구간일 것
7. 최근 1개월 수익률이 -10%보다 양호할 것
8. 현재가가 20일선 또는 50일선 위로 회복했을 것
9. 주도 섹터에 포함되면 가산점을 줄 것

감점 조건은 다음과 같습니다.

- 52주 고점 대비 50% 이상 하락
- 최근 1개월 수익률 -10% 이하
- 상승여력이 90%를 넘는 과도한 목표가 괴리

#### 점수화 개념

```text
성장저평가점수 =
  밸류에이션점수
+ 성장성점수
+ 목표가상승여력점수
+ 최근수급점수
+ 이평선회복점수
+ 유동성점수
- 과도한소외패널티
- 과열/과도한목표가괴리패널티
```

현재 구현은 `main.py`의 `find_undervalued_turnaround_stocks()`와 `scan_us_tickers()`에서 이 기준을 사용합니다.

### 2.3 🌐 사용자 인터페이스 (자바스크립트 / `index.html`)

- **역할**: `data.json` 데이터 로드, 테마별 그룹화, 프리미엄 UI 렌더링.
- **기능**:
  - 각 테마 내 종목 **순위(#1, #2...)** 자동 부여 및 표시.
  - 필터 탭 제공: **[국내 테마]**, **[국내 저평가]**, **[미국 S&P500]**, **[미국 나스닥]**, **[Russell 1000]** 지원.
  - **테마별 그룹화**: 저평가 종목도 원래 어떤 테마(AI, 로봇 등)에서 발굴되었는지에 따라 그룹화되어 표시됩니다.
  - **1개월 수익률 배지**: 종목명 옆에 상승(빨간색), 하락(파란색) 추세를 직관적으로 표시합니다.
  - **프리미엄 카드**: 적정가, 상승여력, 투자 등급 등을 시각적으로 수려하게 표현.
  - 종목별 **투자 지표(PER, PBR, DIV)** 하단 바 렌더링.
- **핵심 파일**:
  - [index.html](/stock/index.html): 메인 구조
  - [style.css](/stock/style.css): 프리미엄 디자인 스타일
  - [script.js](/stock/script.js): 동적 렌더링 로직

## 3. 데이터 스키마 (data.json)

`main.py`가 생성하고 `index.html`이 소비하는 데이터 표준 형식입니다.

```json
[
  {
    "theme": "테마명 (예: 국내 저평가 - 5G)",
    "name": "종목명 (예: LG전자)",
    "code": "종목코드 (예: 066570)",
    "volume": 143400,
    "change_1m": "+20.95%",
    "per": "26.80",
    "pbr": "1.07",
    "dividend": "2.5% 또는 N/A",
    "upside": "+25.40% 또는 N/A",
    "fair_value": "130000 또는 N/A",
    "current_price": 102000,
    "opinion": "4.00매수 또는 N/A",
    "grades": {
      "profit": "최고/우수/보통/주의",
      "health": "최고/우수/보통/주의",
      "growth": "최고/우수/보통/주의"
    },
    "is_profitable": "Pass (흑자) | Fail (적자) | Skipped (API 키 없음)",
    "time": "2026-05-05 10:36"
  }
]
```

### 3.1 category 값 정의

`category`는 화면 필터와 데이터 분류의 기준입니다. 새로운 카테고리를 추가할 때는 `main.py`, `script.js`, `GUIDE.md`를 함께 수정해야 합니다.

| 값 | 의미 | 화면 필터 |
| :--- | :--- | :--- |
| `domestic_theme` | 국내 주도 테마 종목 | 국내 테마 |
| `domestic_value` | 국내 성장 저평가 종목 | 국내 저평가 |
| `us_sp` | 미국 S&P500 성장 저평가 종목 | 미국 S&P500 |
| `us_ndq` | 미국 나스닥 성장 저평가 종목 | 미국 나스닥 |
| `us_rsl` | 미국 Russell 1000 성장 저평가 종목 | 미국 Russell 1000 |

## 5. 개발 및 수정 가이드 (AI 지침)

### ⚠️ 변경 및 협업 원칙

1.  **하드코딩 금지**: `index.html`에 종목 데이터를 직접 입력하지 마십시오. 모든 데이터는 `data.json`을 통해야 합니다.
2.  **구조 유지**: `main.py`는 데이터 생성 및 저장에만 집중해야 하며, HTML 태그를 생성하지 마십시오.
3.  **가이드 자동 업데이트 (필수)**: 새로운 기능 추가, 로직 수정, 또는 데이터 스키마 변경 시, **반드시 이 `GUIDE.md` 파일에도 해당 내용을 최신화**해야 합니다. (모든 수정과 추가된 내용은 가이드에 즉시 업데이트하는 것이 원칙입니다.)
4.  **한국어 우선 원칙**: 프로젝트 내의 모든 문서(`.md` 파일), 주석, 사용자 가이드는 **반드시 한국어로 작성**해야 합니다.

### ✅ 확장 및 고도화 가이드

- **데이터 엔진 확장 시**: `main.py`의 `results` 리스트에 새로운 필드를 추가하면 `data.json`에 자동 포함됩니다.
- **UI 개선 시**: `index.html`의 CSS 변수를 활용하십시오. 시각적으로 'Premium'하고 'Alive'한 느낌을 주기 위해 미세 애니메이션 적용을 권장합니다.
- **로컬 테스트**: `python main.py` 실행 후 `python -m http.server`를 통해 데이터를 확인하십시오.

### 🧭 AI 작업 기본 순서

AI 어시스턴트는 작업을 시작할 때 아래 순서를 기본으로 따릅니다.

1. `docs/GUIDE.md`를 먼저 읽고 현재 기준을 확인합니다.
2. 데이터 수집, 선별 기준, 스키마 변경이면 `main.py`를 수정합니다.
3. 화면 표시, 필터, 정렬, 카드 구성 변경이면 `script.js`를 수정합니다.
4. 레이아웃, 색상, 간격, 반응형 변경이면 `style.css`를 수정합니다.
5. 데이터 스키마나 운영 기준이 바뀌면 `docs/GUIDE.md`도 함께 최신화합니다.
6. 가능하면 `python main.py`로 `data.json`을 재생성합니다.
7. `python -m json.tool data.json`으로 JSON 유효성을 확인합니다.
8. `python -m http.server`로 로컬 화면을 확인합니다.

### 🛠️ 수정 범위 판단 기준

- **값이 없거나 틀린 경우**: `main.py`의 수집/계산 로직과 `data.json`의 실제 필드를 확인합니다.
- **화면에 표시되지 않는 경우**: `script.js`의 렌더링 로직과 데이터 키 이름이 일치하는지 확인합니다.
- **필터 탭이 이상한 경우**: `script.js`의 `filterData()`와 `category` 값을 확인합니다.
- **카드 개수, 너비, 반응형이 이상한 경우**: `style.css`의 `.dashboard-grid`, `.theme-card`, `@media` 규칙을 확인합니다.
- **데이터 스키마가 바뀐 경우**: `main.py`, `data.json`, `script.js`, `docs/GUIDE.md`를 함께 확인합니다.

### ⚠️ 외부 수집 실패 주의사항

`main.py`는 네이버 금융, KRX, Yahoo Finance, Open DART 등 외부 데이터에 의존합니다. 외부 사이트 응답, 차단, 인코딩, 인증 상태에 따라 실행 결과가 달라질 수 있습니다.

- `python main.py` 실행 후 데이터 수가 비정상적으로 줄면 즉시 정상으로 간주하지 말고 원인을 확인합니다.
- KRX 투자지표 수집 실패 시 네이버 상세 데이터 fallback이 사용될 수 있습니다.
- `KRX_ID`, `KRX_PW`, `DART_API_KEY`가 없으면 일부 검증 또는 수집이 제한될 수 있습니다.
- 재생성된 `data.json`이 기존보다 크게 줄었거나 특정 카테고리가 사라졌다면, 배포 또는 커밋 전에 `git diff -- data.json`으로 변경 폭을 확인합니다.
- 외부 수집 실패가 의심되면 코드 변경과 데이터 변경을 분리해서 판단합니다.

## 6. 로컬 실행 및 배포

### 6.1 사전 준비 (Python 설치)

데이터 엔진(`main.py`)을 실행하려면 파이썬 설치가 필요합니다. (프론트엔드 UI 디자인 및 기능 수정만 할 경우에는 설치하지 않아도 됩니다.)

1. [Python 공식 다운로드](https://www.python.org/downloads/)에서 **Python 3.11 또는 3.12** 버전을 다운로드합니다.
2. 설치 마법사 첫 화면 하단의 **`Add python.exe to PATH`** 체크박스를 반드시 체크한 후 설치하십시오. (체크하지 않으면 명령어가 동작하지 않습니다.)
3. python --version 확인

### 6.2 로컬 실행

1. `pip install -r requirements.txt` 또는 `python -m pip install -r requirements.txt`
2. `python main.py` (`data.json` 데이터가 생성됨)
3. `python -m http.server` 실행 후 `localhost:8000` 접속

### 6.3 배포

- GitHub Actions가 주기적으로 `main.py`를 자동 실행하여 최신 데이터가 포함된 `data.json`과 함께 GitHub Pages로 정적 배포를 수행합니다.
