# 주식 투자 지표(PER/PBR) 및 수익성 데이터 누락 수정 결과

대시보드에서 `PER`, `PBR`, `DIV` 및 `is_profitable` (흑자 여부) 데이터가 "N/A" 또는 "Skipped"로 표시되는 문제를 성공적으로 해결했습니다.

## 주요 원인
- **pykrx API 연결 문제**: `pykrx` 라이브러리가 최신 KRX 데이터를 가져오려 할 때 필드 매핑 문제로 오류가 발생하고 있었습니다.
- **DART API Key 부재**: Open DART API 키가 환경 변수에 설정되어 있지 않아 흑자/적자 여부를 검증하는 로직이 스킵되었습니다.

## 조치 사항

### 1. `main.py`에 강력한 스크래핑 Fallback 추가
- `pykrx` 또는 `dart-fss` API가 정상적으로 동작하지 않을 경우를 대비해 **네이버 금융 스크래퍼** (`get_naver_financials`)를 추가했습니다.
- 종목별 네이버 금융 페이지(예: `https://finance.naver.com/item/main.naver?code=...`)의 HTML 구조를 분석해여 해당 종목의 `PER`, `PBR`, `배당수익률(DIV)`을 스크래핑합니다.
- '기업실적분석' 표에서 최근 영업이익 데이터를 추출하여 흑자(`Pass`) / 적자(`Fail`) 여부를 자체적으로 판단할 수 있게 구현했습니다.

### 2. 예외 처리 강화
- 로직 수행 중 `pykrx`가 `N/A`를 반환할 때만 네이버 스크래퍼가 마지막 수단으로 호출되도록 하여, 원래의 속도나 의도를 해치지 않고 정보 복원력을 갖췄습니다.

### 3. 가이드 점검 및 업데이트
- **`doc/guide.md`**의 *2.1 데이터 엔진* 항목에 Fallback (대체 수단) 스크래핑 기능이 추가되었음을 사용자 가이드에 명시했습니다.

## 검증 내역
- 테스트 스크립트를 통해 `main.py`를 수동으로 실행 (`python main.py`)한 결과, 종전 N/A로 뜨던 LG에너지솔루션 등의 `per`, `pbr` 데이터가 정상 값(예: 80.86, 5.53)으로 반환되었음을 `data.json`에서 확인했습니다.
- `이익 여부` 역시 DART API 없이도 네이버 실적 테이블을 참조해 `Pass (흑자)` 또는 `Fail (적자)`가 정확하게 반영되었습니다.
